Project Overview:

This project aims to develop a web-based application that enables users to gather business leads from Google Maps based on specified criteria, such as location and business type. The application will allow users to enter their search criteria, generate a list of leads in real-time, display the data in a table format, and provide the option to download the data as a CSV file. Using Python for data scraping and backend functionality, the system will provide a streamlined solution for users needing targeted business lead information.

Objectives:

  • To create a user-friendly web interface for collecting business leads based on user-defined criteria.
  • To enable real-time data collection and visualization of business information from Google Maps.
  • To provide downloadable CSV files for users to utilize the data for further analysis or outreach.

Scope of Work:

User Interface (Frontend)

  1. Design a simple, responsive HTML form where users can input the location and business type.
  2. Display the generated list of leads in a clean, tabular format.
  3. Provide a download button to allow users to retrieve the data in CSV format.
  4. Style the frontend using Bootstrap or other CSS frameworks to ensure a professional and easy-to-navigate interface.

Backend (Python and Flask)

  1. Set up a Flask application to handle form submission, lead generation, data storage, and CSV downloads.
  2. Use Python libraries such as requests, BeautifulSoup, or Selenium for data scraping (as permitted) or leverage Google Maps API where feasible.
  3. Store generated leads in a CSV file, which will be accessible through the web interface.

CSV Generation and Download

  1. Integrate a CSV file generation feature that saves lead data in a structured format.
  2. Enable a Flask route for secure downloading of the CSV file by users.

Tools and Technologies:

  • Languages: Python, HTML, CSS, JavaScript, GO
  • Frameworks: Flask for backend development
  • Libraries: GO libraries and precompiled programs for scrapping.

Scraper Details:

The scraper accepts three key inputs:


  • Category: Specifies the field or industry from which you need leads.
  • Location: Defines the geographical area for lead extraction.
  • Timeout: Sets the duration for which the scraper will run.

Once the inputs are provided, the scraper executes the script and processes data for the specified duration. During this time, the user must wait until the operation is complete. After the timeout expires, a table displaying the extracted leads and their details will appear on the right side of the page. Additionally, a download button will be available, allowing users to save the results in a file format of their choice.
